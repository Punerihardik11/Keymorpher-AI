## Use Case Diagram Explanation

The Use Case Diagram of the **Keymorpher AI System** illustrates the functional requirements of the system and the interactions between the user and the system.

### Actor
- **User**
  - The primary actor who interacts with the gesture-based virtual keyboard system.

### Use Cases
- **Start Camera**
  - Allows the user to initiate the webcam for capturing hand movements.

- **Detect Hand**
  - The system detects the user’s hand from the camera input.

- **Track Finger Movement**
  - Tracks finger movement to identify gestures used for key selection.

- **Display Virtual Keyboard**
  - Displays a virtual keyboard interface on the screen for user interaction.

- **Detect Key Selection**
  - Identifies which virtual key is selected based on finger position and gesture.

- **Register Key Press**
  - Registers the selected key as a valid key press.

- **Display Typed Text**
  - Displays the text formed by the registered key presses.

- **Clear/Edit Text**
  - Allows the user to clear or edit the typed text.

- **Exit System**
  - Enables the user to safely terminate the system.

### System Boundary
All use cases are enclosed within the **Keymorpher AI System** boundary, clearly defining the scope of the system. The diagram shows how the user interacts with the system to perform gesture-based typing operations.

This use case diagram provides a high-level view of system functionality and serves as the foundation for the OOSE analysis and class diagrams.
